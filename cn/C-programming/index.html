  • 如果分配成功，malloc 返回指向分配的内存块的指针。
  • 如果分配失败（例如，因为系统内存不足），则返回 NULL。

注意：分配的内存块是未初始化的，即其内容是不确定的。如果你需要分配并初始化内存，可以使用 calloc 函数。

它比malloc要多一个参数。但是实际上表示的意思是一样的。

2. Macro宏

你可以使用宏来创建类似于模板的代码，但这通常会牺牲代码的可读性和维护性。宏可以在编译时展开，从而为不同的数据类型生成特定的代码。

虽然宏可以提供一定程度的灵活性，但它们也有一些缺点，比如容易引起代码的不可读性和调试困难。因此，在实际开发中应谨慎使用宏。

5.2 pthread_cond_wait(cond, mutex)

pthread_cond_wait 函数用于等待一个条件变量被触发。它通常与互斥锁一起使用，在调用这个函数之前，线程应该已经获得了与条件变量相关联的互斥锁。函数的效果如下：

  1. 释放互斥锁：当线程调用 pthread_cond_wait 时，它会自动释放与条件变量相关联的互斥锁，以便其他线程可以获得锁并改变条件变量的状态。
  2. 等待条件变量：线程会被阻塞，等待直到其他线程调用 pthread_cond_signal 或 pthread_cond_broadcast 来触发条件变量。这意味着线程会一直等待，直到条件变量被满足。
  3. 重新获取互斥锁：一旦条件变量被触发，线程会被唤醒并自动重新获取之前释放的互斥锁。然后线程可以继续执行后续的操作。

使用 pthread_cond_wait 可以有效地实现线程间的同步，避免忙等待（busy waiting），从而提高程序的效率和响应性。在生产者-消费者问题中，这个函数通常用于等待缓冲区非空或非满的条件，以实现正确的生产和消费行为。

疑问：既然线程会阻塞，那为什么在外面要使用while来判断，而不是if？

使用 while 循环来判断条件而不是 if 语句是为了防止所谓的“假唤醒”（spurious wakeups）和确保条件确实满足。

假唤醒

假唤醒是指线程可能在没有接收到明确的条件变量信号的情况下被唤醒。这是由于操作系统或者硬件层面的优化和实现细节导致的。如果使用 if 语句，线程在被假唤醒时会继续执行后续的代码，即使条件并没有真正满足。这可能导致程序逻辑出错。

使用 while 循环可以确保在这种情况下线程会再次检查条件，如果条件不满足，线程会继续等待。

使用pthread_cond_signal(cond)来唤醒一个阻塞的线程。
