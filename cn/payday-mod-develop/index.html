首先是对mod.txt的解读。

mod.txt主要有以下几个部分：

  • 前置内容，包括模组名称、模组信息、模组cover等信息
  • keybinds: 用于进行按键绑定的设置，有如下内容：
  • persist_scripts: 持续运作的脚本文件，以下是官方的解释：

A script in script_path which is run every frame until the global variable specified in global is set to anything other than false or nil.

  • persist_scripts: The array of objects containing a global and a script_path.
  • global: The global value to associate with script_path.
  • script_path The script to continuously run until the global variable global is set to a value other than false or nil.

不难看见每一个persist_script的最下方有一个标识，这个标识往往被设置为True，就比如UTClassUT = true，UTMenuMain = true。这个标识就是要被放在global字段里面的值，mod loader也会根据这个来判断是否要执行这个script。

我推测，可能是先进行hook，然后才加载persist_scripts。

注意：

需要注册的任何类、变量、函数，都可以通过hook的方式来进行。不是说把脚本设置成persist_scripts就可以一直使用了，那还是有不可使用的情况。正确的理解应该是，hook进原函数了之后，才可以完成注册，才可以在后续进行使用。以前的理解是错误的。

UT类的生存周期

Lua在执行的过程中，变量就只有全局变量和局部变量之分。我们看到UTClassUT被设置为True，这就表示那个脚本会一直运行，也就表示那个脚本里面的UT类会一直存在。

所以，在其他的脚本中，有直接对UT类进行操作的，这就是因为UT已经通过persist_script，成了全局变量，不需要再import什么的，直接使用即可。

从更新的理解来看，是因为UT类所在的script被设置为了persist_script，也就是一个可以持续运转的脚本，所以这个UT才会一直存在在生存周期中。如果说没有设置这个的话，估计也无法直接调用UT。

4. 有关几种call方案

官方文档：

  • Hooks: https://payday-2-blt-docs.readthedocs.io/en/latest/lua/hooks/
  • DelayedCalls: https://payday-2-blt-docs.readthedocs.io/en/latest/lua/delayed_calls/

这个DelayedCalls:Add()函数非常有用。在这里节选三个比较常用的函数：

DelayedCalls:Add( id, time, func )

Adds a delayed call that will automatically be ran after the time has expired.

  • id The unique identifier of this delayed call.
  • time A time value, in seconds, that the function func should run after.
  • func The function that should be ran if time passes.

有关这个delayedcall，我发现它实际上是开了一个协程在进行操作的，因为下一句马上就打印出来了，而没有等待2.5s。所以推测它的底层应该是协程函数。

Hooks:PreHook( object, func, id, pre_call )

Automatically hooks a function to be called before the specified function on a specified object.

  • object The object for the pre-hook to search for the func to hook.
  • func A function name, as a string, on the object to be pre-hooked.
  • id The unique identifier for this pre-hook.
  • pre_call The function that should be called before func is called on object.

也就是说，这个就是作用在某一个类的某一个函数里面的，pre_call才是放进去自己写的函数。
