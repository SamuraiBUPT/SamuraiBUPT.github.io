1. C++ 面向对象

1.1 类间的链接设计

在开发过程中，注意到A类中有B类作为成员变量，而B类中又有A类作为成员变量。并且在赋值的时候也是相互赋值。

这样的设计思路，可以理解为两个大类之间，有一个桥梁，可以连接你我、触及你我。在实际的代码开发中，经常需要两个类互相调度使用资源，这个时候就有了这样的相互式的、反向的指针，指回对方类，以控制对方资源。

案例 :

TritonModel掌握的资源：

  • server

    • rate limiter
    • payload

  • scheduler

    • TritonModel （可以反向获得那个TritonModel资源），在初始化scheduler的时候，就是反向绑定了自己，传入了this。

    • scheduler通过成员变量“curr_payload"进而控制requests，而curr_payload_的初始化又是通过调用成员变量TritonModel，反向获得了Model的payload。

这些类中的互相指向关系，赋值的方法有多种：

  • 在初始化的时候就进行赋值与绑定
  • Register()成员函数，可以负责进行赋值与绑定。

1.2 从OnRelease()谈RAII

在代码开发过程中，我经常看到一些类的 状态切换方法 ，诸如 reset, Release, OnRelease等等，下面对这些方法的设计思想进行探索。

  1. OnRelease()：这种命名可能不如其他两个常见，但它通常是一个回调或虚函数，当资源被释放时会被调用。例如，在某些自定义引用计数对象中，当引用计数降为0时，OnRelease()可能被调用。
  2. reset()：
    • 在C++的智能指针如std::shared_ptr和std::unique_ptr中，reset()方法会释放其当前指向的对象，并使指针指向nullptr或新的资源。
    • 用于将对象状态重置为初始状态或释放其内部资源。
  3. Release()：
    • 在COM（Component Object Model）编程中，接口有一个AddRef()和一个Release()方法，用于管理对象的引用计数。当Release()被调用并导致引用计数减少到0时，对象会自动销毁。
    • 在其他上下文中，Release()方法可能用于释放类所持有的资源，例如内存或文件句柄。

这些函数的核心设计思想是资源管理。在C++和其他需要手动或半自动资源管理的语言中，正确地分配和释放资源是非常重要的。不正确地管理资源可能导致内存泄漏、文件描述符泄漏或其他类型的资源泄漏。

这种编程思想与RAII（Resource Acquisition Is Initialization）紧密相关，这是C++中的一个关键概念，意味着资源的获取（例如，分配内存、打开文件等）与对象的初始化紧密绑定，资源的释放与对象的销毁紧密绑定。

总之，当你看到这些函数时，应该立即意识到它们与资源管理、对象生命周期和可能的引用计数有关。如果你正在使用或设计这样的类，确保你了解其资源管理策略，以防止资源泄漏或其他问题。

1.3 单例模式

在这个类里面，它的构造函数被声明为private，然后后面有一个static修饰的create函数。

这种设计模式通常被称为“工厂方法”模式或“单例模式”的一种变种。通过使用静态的Create方法来创建类的实例，并将构造函数声明为私有，这种设计实现了对对象创建过程的严格控制。

我们可以观察到，Create方法最后的参数指针，为unique指针。

我们可以深入Create方法里面去看看：

  • 首先是调用自己的构造函数。通过new分配堆空间。
  • 然后调用unique_ptr掌管这一部分堆空间的内存。
  • 最后把这个unique_ptr的所有权释放，赋值给参数的最后一个，也就是Output，scheduler也经过reset方法对这个申请的资源进行持有。
  • 这样的流程保证了单例。

why static ?

static关键字用于类的成员函数表示这个函数不依赖于类的具体实例。也就是说，一个static成员函数可以在没有创建类的对象的情况下被调用。
