1.1 let关键字

原来的var声明变量有很大的不好之处。

  • let声明的变量不能重复定义。如果之前用了let定义这个变量，之后无法再次定义
  • let变量具有作用域。而不是像var一样，在哪里声明了都可以用，无论是代码段的内部还是外部。
  • let变量不能提升，不能先print再定义。如果是var的话，就可以，这个叫做变量的提升，至于是怎么做到的，之前在pink老师的讲堂上讲过js的运行逻辑，在这里不做赘述。

1.2 const关键字

核心概念

  • 不会改变变量指向的地址
  • 但是可以改变这个地址上赋值的对象

使用const声明常量。

  • 代码习惯：常量一般用大写字母表示
  • 声明了之后必须给予初始值，否则会报编译错误。
  • 原则上，常量是不允许改变赋值的，但是在这里有一个要点：
    • 如果是对一个string或者number改变赋值，会报错。
    • 如果是对一个数组进行操作，比如array.push(30)，并不会报错。
      • 因为在这里并没有改变这个变量的赋值，就是地址并没有变，仅仅是变了元素，这个是可以接受的。
      • 所以，可以对数组或者对象进行操作。

使用 const 声明的变量不能被重新赋值。但这并不意味着其内容是不可变的，特别是当它引用的是对象或数组时。

意思就是，通过const声明的是一个对象，保证这个对象的地址不会更改，而至于这个对象本身的值，并不是不可以更改的。

4. Promise异步编程

Promise 有三种状态：

  1. Pending：初始状态，既不是成功也不是失败。
  2. Fulfilled：意味着操作成功完成。
  3. Rejected：意味着操作失败。

静态方法：

  • Promise.all([]): 等待所有 promises 完成。
  • Promise.race([]): 返回第一个完成（无论是 resolve 还是 reject）的 promise。
  • Promise.resolve(): 返回一个状态为 fulfilled 的 Promise 对象。
  • Promise.reject(): 返回一个状态为 rejected 的 Promise 对象。

5. this指向问题探究

  1. 对象方法中的 this： 当函数作为对象的方法被调用时，this 指向该对象。

  2. 构造函数中的 this： 当使用 new 关键字调用函数（构造函数）时，this 指向新创建的对象实例。

  3. 事件监听器中的 this： 当在 DOM 事件监听器中使用函数时，this 通常指向触发事件的元素。

  4. 箭头函数中的 this： 箭头函数不绑定自己的 this。它们从定义它们的函数上下文继承 this。

    箭头函数体内的this对象，就是定义该函数时所在的作用域指向的对象，作用域是指函数内部，而不是使用时所在的作用域指向的对象。

  5. 手动设置 this： 使用 call(), apply(), 或 bind() 可以明确设置函数调用时的 this 值。

7.4 CommonJS与require()

也许经常看到这样的模块导入：

这个是怎么做到的呢？这就是CommonJS。

在 CommonJS 出现之前，JavaScript 没有官方的模块系统。随着 Node.js 的出现和 JavaScript 的应用场景变得越来越复杂，需要一种组织代码的方法，使得代码更加模块化、更容易管理和维护。CommonJS 模块系统满足了这个需求。

模块导出

提问：使用CommonJS这种模块导入的方式，和使用import和export这种导入导出的方式，有何不同？

CommonJS（主要用于 Node.js）和 ES6 模块（或称为 ECMAScript 模块，简写为 ESM）都是 JavaScript 的模块系统，但它们在设计、使用和目标上有所不同。

  1. 初始化时机：
    • CommonJS：模块在 require 时同步加载、解析和执行。当你调用 require() 函数，模块的代码会被执行，并且导出的值会被返回。
    • ES6 模块：模块是异步解析的，但可以同步或异步加载。它允许更高级的静态分析，如 Tree Shaking（摇树优化）和动态 import()。
  2. 导入的值：
    • CommonJS：require 导入的是一个值的拷贝或引用，这取决于导出的值的类型。
    • ES6 模块：导入的始终是一个引用，而不是一个拷贝。当导入的值在其原始模块中发生变化时，这种变化会反映到所有导入了它的模块。
  3. 语法不同。
  4. 动态导入：
    • CommonJS：你可以在运行时根据条件动态地使用 require。
    • ES6 模块：虽然基本的 import 语法是静态的，但你可以使用 import() 语法动态导入模块。
  5. 其他差异：
    • CommonJS：有一个全局的 module 对象。
    • ES6 模块：有一个静态的导入和导出结构，这意味着你不能在一个模块内部的条件语句中使用 import 或 export。
  6. 兼容性：
    • CommonJS：在 Node.js 的早期版本中广泛使用。
    • ES6 模块：现代浏览器支持，Node.js 也在较新的版本中提供了支持（通常需要文件扩展名 .mjs 或配置 "type": "module" 在 package.json 中）。
  7. 用途：
    • CommonJS：最初为 Node.js 设计，因此在服务器端代码中更为常见。
    • ES6 模块：设计为静态分析友好，主要用于前端，但也逐渐在 Node.js 中得到应用。

Appendix: Syntax

1. 等号

双等号 (==) - 宽松相等 (Loose Equality)

  • 当使用 == 进行比较时，如果两个值的类型不同，JavaScript 会尝试自动进行类型转换以使其相等。
  • 一旦进行了类型转换并且值相等，则返回 true。

三等号 (===) - 严格相等 (Strict Equality)

  • 当使用 === 进行比较时，不仅值要相等，类型也必须相等。
  • 不会发生自动的类型转换。

建议

  • 通常建议在 JavaScript 中使用三等号 (===) 进行比较，因为这可以避免意外的类型转换导致的错误。
  • 如果你明确知道需要进行类型转换，并希望 JavaScript 帮你完成，那么可以使用双等号 (==)。
